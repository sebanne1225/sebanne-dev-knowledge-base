# Unity / VRChat ツール UI 設計の共通前提

## 目的
今後の Unity / VRChat 向けツール UI で、  
毎回ゼロから見た目や操作方針を考え直さないための共通前提。

## 基本方針
- Unity 標準 Inspector に近い見た目を優先する
- カード風の強い演出より、実務向けで軽い一覧 UI を優先する
- 「何のオブジェクトか」「結果がどうか」がすぐ分かる構造を優先する
- 独自操作を増やすより、Unity のいつもの触り方で使えることを重視する

## 情報の出し方
- 主情報と補助情報を分ける
- 一覧では主情報を優先し、詳細は薄くするか折りたたむ
- 一覧性を崩す長文説明や大きすぎる行は避ける
- 結果、理由、補正予定は全部同じ強さで並べず、優先度を付ける
- 理由や次アクションが弱い分類は、独立カテゴリとして無理に残さない

## 操作設計
- 専用ボタンを増やす前に、Unity 標準操作で足りるか確認する
- `Select` や `Ping` のような専用ボタンは、標準の `ObjectField` 操作で代替できるなら増やさない
- 手入力より、`ObjectField` / Drag&Drop / 自動補完 / 自動反映を優先する
- 内部では path 保持でもよいが、入力はオブジェクト選択ベースを優先する
- 一覧 UI では、その場で完結できる軽い操作を優先し、重い編集は Settings 側へ残す

## ウィンドウ構成
- EditorWindow は操作フローに沿ったセクションブロック構成にする
  （例: 入力設定 → 出力設定 → 実行）
- 各セクションを `EditorGUILayout.BeginVertical(EditorStyles.helpBox)` で囲み、
  視覚的にブロック分けする
- セクション見出しは `LabelField(boldLabel)` + miniLabel 補助文 1 行を枠内の先頭に置く
- 普段触らない設定（上級設定等）は最下部に Foldout で折りたたむ
  - Foldout は枠の外に置き、展開時の中身だけ helpBox 枠で囲む
- 関連する一覧（スロット一覧等）は選択 UI の直下に配置する
- ウィンドウ全体を ScrollView で囲む
- パス表示はラベル付き（例:「現在の出力先」）で存在感を持たせる。
  gray miniLabel だけだと見落としやすい

## 結果表示
- 診断結果は一時的なウィンドウ状態だけに持たせず、再確認しやすい場所へ保持する
- 再生や再読み込みで結果がすぐ消える作りは避ける
- 「あとで見返せること」「どの対象の結果か分かること」を重視する
- 実体の再検出と、既存結果への再評価は分けて扱い、軽い再評価は一覧側で回し、重い再検出は明示操作に寄せる

## 説明と警告
- `HelpBox` は注意喚起と判断材料に使い、単なる説明には使わない
  - Warning: 不正入力、未導入、範囲超過、上書き確認
  - Error: 操作をブロックする致命的条件
  - Info: モード説明など「選択の判断材料になる情報」に限定
- 単なる説明・計算結果・プレビュー情報は miniLabel スタイルの補助文で出す
- 説明用 `HelpBox` の連続表示は避ける
- セクションごとに短い説明は歓迎だが、説明の圧が強すぎないようにする
- 情報表示の HelpBox が 3 つ以上連続する場合は miniLabel への降格を検討する

## 用語方針
- 日本語優先で分かりやすくする
- ただし Unity / VRChat / コンポーネント固有名は英語を残す
- ツール側の概念語は日本語中心でよい
- 英語併記は必要最低限にし、固有名詞や元コンポーネント名に限定する
- Dry Run / Generate のような操作系の英語は、Unity や開発文化で定着しているものはそのまま残してよい

## 例外設定
- 基本は全体ルールで回る UI を優先する
- ただし、特殊ケースを逃がすための個別設定は必要
- 「例外設定はあるが、例外だらけにはしない」設計を優先する
- 個別設定の意味は短い説明でその場で分かるようにする

## 好みの UI 傾向まとめ

### 好むもの
- Unity 標準っぽい見た目
- コンパクトで一覧性が高い UI
- 情報の強弱がある UI
- 入力ミスしにくい UI
- 結果が残って再確認しやすい UI
- 日本語優先だが固有名詞は残す表記
- セクションごとに helpBox 枠でブロック分けされた構成

### 好まないもの
- 大きすぎるカード UI
- 独自ボタンや独自操作が多い UI
- 説明が前に出すぎる UI
- 手入力前提の UI
- 一時的で消える結果表示
- すべての情報が同じ強さで並ぶ UI
- 無理に全部日本語化した表記

## ひとことで言うと
「賢そうな UI」より、「迷わず使えて、あとで見返しても崩れない UI」を優先する。
