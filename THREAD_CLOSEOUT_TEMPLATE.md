# THREAD_CLOSEOUT_TEMPLATE

## 目的
スレ終了時に、そのスレで出た内容の反映先と内容を決めてから、
まとめて反映するためのテンプレ。

核心: **「決める」と「書く」を分ける。**
全部決めてから、まとめて書く。書きながら決めない。

## このテンプレでまだやらないこと
- AGENTS / config の自動更新
- Skill 実装や config 追加
- 単発ログの大量保存

## 抽出ルール
- 推測で埋めない
- 単発ログより、今後も再利用する内容を優先する
- 共通化向き / repo 固有 / 見送り を分ける
- CLAUDE.md 候補は 3〜5 行で短くする

---

## closeout フロー

### Phase 1: 棚卸し
※ リポ変更がないセッション（Notion 整理のみ等）では step 1-2 をスキップし step 3 から始める。

1. □ 網羅的実装確認（Claude Code に依頼）
2. □ 修正が出たら修正依頼
3. □ 会話中の 📝 メモを全件抽出する（経緯セクションの骨格になる）
4. □ Claude Code に変更ファイル一覧＋要約を出させる（やったことセクションの素材 + Notion 整合チェックの入力）
5. □ Notion 整合チェック（Claude が実施）
   - Claude Code から受け取ったリポの事実（version, 実装済み機能, CLAUDE.md 状態）と Notion 3DB の関連行を突合
   - リリース管理: version・ステータスがリポ実態と一致するか、重複行がないか
   - 次フェーズ候補: 今回実装したものが「完了」になっているか、新しい後回しが漏れていないか、CLAUDE.md の次フェーズ候補欄とズレていないか
   - セッション記録: 過去の記録と今回の内容に矛盾がないか
   - 矛盾があれば Phase 2 の反映リストに含める

### Phase 2: 全部決める（ファイルも Notion も一切触らない）
6. □ セッション記録の内容を決める — 以下の 4 セクション構成:
   - やったこと: Phase 1 の Claude Code 変更要約を叩き台にする。Claude の記憶だけで書かない
   - 経緯: Phase 1 で抽出した 📝 メモを骨格にし「せ・C・→決定」形式で再構成する。📝 がカバーしてない判断ポイントは会話から補完
   - 後回し: 📝 に含まれる後回し判断 + 次フェーズ候補 DB との突合
   - 引き継ぎ: 次スレ開始テンプレにそのまま貼れる粒度。品質テスト：「別の Claude がこれだけで動けるか」
7. □ 反映リストを作る（下記テンプレ）
8. □ せばんぬ確認 →「これで全部」

### Phase 3: まとめて反映（確認済みのリストを消化するだけ）
9. □ Claude Code にファイル更新を一括依頼
10. □ Claude が Notion に投入
11. □ 変更ファイル一覧を出す
12. □ 反映確認
    - Claude Code: 変更ファイルの内容が反映リストと一致するか確認
    - Claude: Notion 投入結果を反映リストと突合
    - せばんぬ向け: 手順欄にコピペが必要なファイル一覧を明示

Phase 2 の確認が通るまで Phase 3 に入らない。
Phase 3 では判断しない。反映リスト通りに実行するだけ。
Phase 3 の全ステップが完了するまでスレを閉じない。

### Phase 3 完了後（せばんぬ手作業）
- [ ] セッション記録に会話リンクを貼る
- [ ] PROJECT_INSTRUCTIONS.md に変更があった場合、プロジェクト手順欄にコピペ

## 反映リスト テンプレ

Phase 2 で作る。せばんぬに見せて「これで全部」をもらうための一覧。

```
## 反映リスト

### セッション記録（Notion 投入内容）
※ 投入時に「日付」プロパティを必ずセットする（空欄になりやすい）
- やったこと: （Claude Code の変更要約をベースに箇条書き）
- 経緯: （会話形式。判断の流れが追える粒度で書く）
  - せ: ○○はどうする？
  - C: △△の理由で□□がよい
  - → □□に決定
- 後回し: （あれば。次フェーズ候補 DB への投入内容と一致させる）
- 引き継ぎ: （次スレ開始テンプレの形式で埋め込み）

### ファイル（Claude Code に一括依頼）
- [ ] {repo} CLAUDE.md: {何を更新するか}
- [ ] knowledge base {ファイル名}: {何を追記/変更するか}
- [ ] REPO_INDEX.md: 変更なし / {変更内容}

### Notion
- [ ] セッション記録 DB: 本セッションの記録投入
- [ ] 次フェーズ候補 DB: {追加/変更内容} / 変更なし
  ※ 後回しを投入する時は、今回のセッション記録ページを「関連セッション」リレーションに必ずセットする
- [ ] リリース管理 DB: {更新内容} / 変更なし

### 見送り
- {内容}: {見送る理由}
```

「変更なし」も明示する。書かないと「忘れたのか、不要なのか」が分からない。

## 次スレ開始テンプレ

closeout 後、次スレを始める時はこのテンプレをそのまま冒頭に貼る。

※ 冒頭に引き継ぎパッケージがないと、状況確認のやり取りが発生して
スレの立ち上がりが遅くなります。

※ 次スレ開始時、Claude は必要に応じて Notion（次フェーズ候補 DB・リリース管理 DB・セッション記録 DB）を読んで現在の状況を確認してから始める。

```
## 前スレでやったこと
（箇条書き3〜5行）

## 現在の状態
（CLAUDE.md の Current State 要約）

## 今日やりたいこと
（1〜3行）

## 引き継ぎ
- repo: （対象 repo のパス）
- CLAUDE.md（repo 固有 / knowledge base）を読んでから始めてください
- 前回の closeout 候補の反映: 済み（※「済み」以外は不可。反映は closeout 内で全て完了させてからスレを閉じる）
```

### 引き継ぎの作り方

引き継ぎはセッション記録の一部として作る。
「次スレの Claude が冒頭で何をするか」をイメージして構成する。
次スレの Claude が Notion を読む必要があるか、どの CLAUDE.md を読むか、最初のアクションは何かが、引き継ぎセクションを見ただけで分かる状態にする。

closeout 後、Claude（claude.ai）が草案を出したら
以下の指示を Claude Code に渡して整形・補完してもらう。

```
セッション記録の引き継ぎセクションを作ってください。

## 読んでほしいファイル
- sebanne-dev-knowledge-base の CLAUDE.md 全体
- 対象 repo の CLAUDE.md

## 出力してほしいもの
THREAD_CLOSEOUT_TEMPLATE.md の「次スレ開始テンプレ」形式で
セッション記録の引き継ぎセクションに埋め込める状態にしてください。
```
